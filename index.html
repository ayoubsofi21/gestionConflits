🚀 Guide Complet des Commandes Git

🔹 1️⃣ Configuration de Git

Configurer Git (nom, email, éditeur…)

git config --global user.name "Ayoub Sofi"
git config --global user.email "ayoub@email.com"

Définit ton nom et ton email pour tous les commits sur tous les projets (--global). Sans --global, la configuration ne s'applique qu'au dépôt actuel.

git config --list

Affiche toutes les configurations Git actives.

🔹 2️⃣ Initialisation & Clonage

git init

Crée un nouveau dépôt Git dans le dossier actuel et ajoute le dossier caché .git contenant l'historique.

git clone https://github.com/user/project.git

Copie un dépôt distant (ex: GitHub) sur ton PC avec tout l'historique.

🔹 3️⃣ Vérification de l’état

git status

Affiche l’état des fichiers : modifiés, ajoutés, ou non suivis.

git log

Montre l’historique complet des commits.

git log --oneline

Version courte de l’historique, un commit par ligne.

git diff

Montre les différences entre les fichiers modifiés et le dernier commit.

🔹 4️⃣ Ajouter & Commit

git add file.js

Prépare un fichier spécifique pour le prochain commit.

git add .

Prépare tous les fichiers modifiés pour le prochain commit.

git commit -m "Add scoring function"

Crée un commit avec un message descriptif.

git commit -am "Fix bug"

Combine add + commit pour tous les fichiers déjà suivis.

🔹 5️⃣ Branches

git branch

Liste toutes les branches locales.

git branch feature-algorithm

Crée une nouvelle branche appelée feature-algorithm.

git checkout feature-algorithm

Change de branche pour se positionner sur feature-algorithm.

git checkout -b feature-ui

Crée une nouvelle branche et s’y positionne directement.

git switch feature-ui
git switch -c new-branch

Nouvelle façon moderne de changer ou créer+changer de branche.

🔹 6️⃣ Fusion (Merge)

git merge feature-algorithm

Fusionne la branche feature-algorithm dans la branche actuelle. ⚠️ Peut générer des conflits.

🔹 7️⃣ Conflits

CONFLICT (content): Merge conflict

Indique qu’un fichier a été modifié dans deux branches. Supprimer les marqueurs <<<<<<<, =======, >>>>>>> pour corriger le conflit.

git add .
git commit -m "Resolve merge conflict"

Après correction, ajoute et commit le fichier.

🔹 8️⃣ Dépôt distant

git remote -v

Affiche les dépôts distants configurés.

git remote add origin https://github.com/user/project.git

Ajoute un dépôt distant nommé origin.

git push origin main

Envoie les commits vers le dépôt distant.

git pull origin main

Récupère et fusionne les changements distants.

git fetch

Récupère les changements distants sans les fusionner.

🧠 Résumé simple

init → créer repo
clone → copier repo
add → préparer commit
commit → sauvegarder
branch → créer branche
checkout → changer branche
merge → fusionner
push → envoyer
pull → récupérer
reset → annuler
stash → sauvegarde temporaire
